# Android APK Build Guide — Cultivation Workout

## Overview

The application is wrapped in a **Capacitor** native Android shell. The Android WebView loads the web application from the production server (`http://192.168.1.116:3000`), so the server must be running and reachable from the device's network for the app to function.

---

## Prerequisites

| Requirement | Version |
|---|---|
| Node.js | 20+ |
| npm | 10+ |
| Android Studio | Latest (Hedgehog or newer) |
| Android SDK | API 24+ (minimum), API 35 (target) |
| JDK | 17 |

Install Android Studio from <https://developer.android.com/studio>.

```
android/                  ← Native Android project (generated by Capacitor)

```

## Quick Start

### 1. Install dependencies

```bash
npm install
```

### 2. Sync Capacitor

This copies web assets and Capacitor configuration into the Android project:

```bash
npx cap sync android
```

### 3. Open in Android Studio

```bash
npx cap open android
```

This opens the `android/` project in Android Studio.

### 4. Build the APK

#### Debug APK (for testing)

In Android Studio:
1. Select **Build → Build Bundle(s) / APK(s) → Build APK(s)**
2. The debug APK will be generated at:
   ```
   android/app/build/outputs/apk/debug/app-debug.apk
   ```

Or from the command line (requires Android SDK in PATH):

```bash
cd android
./gradlew assembleDebug
```

#### Signed Release APK (for distribution)

1. In Android Studio, select **Build → Generate Signed Bundle / APK**
2. Choose **APK**
3. Create or select a keystore:
   - **Keystore path**: choose a secure location (do NOT commit to Git)
   - **Key alias**: `cultivation-release`
   - Set passwords for both keystore and key
4. Select **release** build variant
5. The signed APK will be generated at:
   ```
   android/app/build/outputs/apk/release/app-release.apk
   ```

Or from the command line, create `android/keystore.properties`:

```properties
storePassword=YOUR_STORE_PASSWORD
keyPassword=YOUR_KEY_PASSWORD
keyAlias=cultivation-release
storeFile=/path/to/your/keystore.jks
```

Then build:

```bash
cd android
./gradlew assembleRelease
```

## Changing the Server URL

Edit [capacitor.config.ts](capacitor.config.ts) and update the `server.url` value:

```ts
server: {
  url: "https://your-production-domain.com",
  cleartext: false, // set to false if using HTTPS
},
```

Then re-sync:

```bash
npx cap sync android
```

## Installing the APK on a Device

### Via USB

```bash
adb install android/app/build/outputs/apk/debug/app-debug.apk
```

### Via File Transfer

1. Copy the `.apk` file to the Android device
2. Open the file on the device
3. Allow installation from unknown sources when prompted
4. Install and launch

## Troubleshooting

### App shows blank screen
- Verify the server is running at the configured URL
- Ensure the device is on the same network as the server
- Check `android:usesCleartextTraffic="true"` is set in `AndroidManifest.xml`

### Connection refused
- Confirm the server IP and port in `capacitor.config.ts`
- If using an emulator, use `10.0.2.2` instead of `localhost`/`127.0.0.1`

### Build fails in Android Studio
- Ensure JDK 17 is configured: **File → Project Structure → SDK Location**
- Run **File → Sync Project with Gradle Files**
- Try **File → Invalidate Caches / Restart**

## Security Notes

- **Never commit** keystore files (`.jks`, `.keystore`) or `keystore.properties` to Git
- When deploying to production, switch from HTTP to HTTPS and set `cleartext: false`
- The `android/` directory `.gitignore` already excludes build outputs and local config
